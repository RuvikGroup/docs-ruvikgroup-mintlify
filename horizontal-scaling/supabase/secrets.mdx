### Required Secrets

You need to set the following secrets for the application to function correctly:

- `SUPABASE_DB_URL`: The connection string for your Supabase database.
- `SUPABASE_URL`: The URL for your Supabase project.
- `SUPABASE_ANON_KEY`: The anonymous key for your Supabase project.
- `SUPABASE_SERVICE_ROLE_KEY`: The service role key for your Supabase project.
- `RC_JUBLEY_PROJECT_ID`: Your RevenueCat project ID.
- `REVENUECAT_API_SECRET`: Your RevenueCat API secret.
- `REVENUECAT_WEBHOOK_SECRET`: The secret for verifying RevenueCat webhooks.
- `ALGOLIA_APP_ID`: Your Algolia Application ID.
- `ALGOLIA_ADMIN_KEY`: Your Algolia Admin API Key.
- `SYNC_ALL_PROFILES_SECRET`: A secret key for a custom function to sync profiles.
- `N8N_RESEND_API_KEY`: API key for Resend, used via n8n workflows.
- `N8N_API_KEY`: API key for your n8n instance.
- `ONE_SIGNAL_APP_ID`: Your OneSignal App ID for push notifications.
- `ONE_SIGNAL_API_KEY`: Your OneSignal REST API Key.

### Adding Secrets

- From you [project dashboard](https://supabase.com/dashboard), go to `Storage` and click on `Edge Functions`.
- Click on `Secrets` on the left panel

![secrets](/images/supabase/secrets-panel.png)

![secrets](/images/supabase/secrets-form.png)

- Add each required secrets, click on `Save` to save each.

### Using Supabase CLI

- Use the Supabase CLI to set secrets for your project.

```bash
supabase secrets set SECRET_NAME=SECRET_VALUE
```

- You can list existing secrets with:

```bash
supabase secrets list
```
