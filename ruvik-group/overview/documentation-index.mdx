## Contributing to Documentation

<Info>
  Help us improve! If you find missing or outdated documentation, please create an issue or submit a PR.
</Info>

### Documentation Standards
1. Use clear, concise language
2. Include code examples where applicable
3. Keep information up-to-date
4. Add troubleshooting sections
5. Cross-reference related documents

### Maintenance Schedule
- **Weekly**: Review and update critical documentation
- **Monthly**: Audit for outdated information
- **Quarterly**: Major documentation review and reorganization
